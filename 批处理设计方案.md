# 批处理设计方案

## 目标
1. 一次批量处理的笔记保存在同一个新文件夹下
2. process 支持批量处理，同时不同来源的输入互不干扰，参数共享。process link1 link2 --args
3. search 之后的批量处理也采用相同的逻辑。
4. 批量处理也可指定目录 search "..." --output-dir ... 或 process ... --output-dir ...

## 设计
- 某个批量执行器：生成批次目录。串行执行。统一错误处理。进度报告。
  - 接收一个待处理列表。
- search 和 process 命令复用 BatchProcessor

直接传递 output_dir 给 generate()：

note_generator.generate(..., output_dir=batch_dir)

## 目录命名
```plaintext
~/.bilinote/data/output/notes/
├── batch_20250420_143052/          # process 批量
│   ├── BV1xx.md
│   └── BV2yy.md
├── search[达奇-40k]_20250420_143100/          # search "达奇 40k"
│   ├── BV3zz.md
│   └── BV4ww.md
└── BV5single.md                    # 单条处理（保持现有）
```

- search 文本要做 slug 化处理。

- --output-dir 仅控制笔记输出位置，缓存位置不变。
  

批量处理完成后，用户可能需要：
查看批次处理摘要（成功/失败数量）
重新处理失败的项
