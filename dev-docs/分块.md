## GPT 分块逻辑优化方案

---

### 一、目标

解决当前分块机制下各 Chunk 独立处理导致的上下文丢失问题，同时将触发条件从字节估算改为更贴合 LLM 实际的 Token 估算。


---

### 二、分块触发条件改造

删除现在的 45MB 分块预算。改用 字符数估计的Token 数作为分块预算。

**文本 Token 估算公式**：
```
token_count = 中文字符数 × 1.8 + 英文字符数 × 0.3
```

**双层配置**：

| 配置项 | 默认值 | 配置归属 | 存放位置 |
|--------|--------|----------|----------|
| `token_limit` | 128000 | 用户配置 | 用户目录的 YAML 配置文件 |
| `token_threshold` | 0.8 | 开发者配置 | `src/config/` 目录下的 JSON 文件 |

**实际分块预算** = `token_limit × token_threshold`。阈值预留空间用于容纳系统 Prompt、历史上下文和模型回复。

> **暂不处理图片 Token**。视频理解模式下图片的 Token 消耗暂不在本次估算范围内，后续若需支持可再扩展。

---

### 三、跨块上下文传递

从第 2 个 Chunk 开始，在请求中附加之前块的总结笔记，使模型掌握前文脉络。

**传递范围**：仅附加**最近 N 个块**的总结笔记，而非全部历史。  
**N 的值**：放入开发者配置，默认 `3`。

**对模型的约束**：  
附加提示词明确告知模型：
- 前面内容是「已生成的历史笔记概要」，仅供参考以保持整体连贯性；
- **不得重复输出**历史内容；
- **仅基于当前块转录文本**生成本段笔记。

> **不做原文重叠**（即不附加上一块结尾的原始转录文本）。

---

### 四、Prompt 内容顺序

每个 Chunk 的请求按以下顺序组装：

1. **系统指令与全局信息**（角色定义、视频标题、标签、风格、格式要求等）
2. **历史上下文区**
   - 最近 N 个块的总结笔记（按时间顺序排列）
   - 说明提示词：明确此为历史概要，禁止重复输出
3. **当前块指令**
   - 约束提示词：要求仅返回本段 Markdown 内容，不生成全局总结
4. **当前块转录文本**（及该块关联的图片）

---

### 五、配置项汇总

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `gpt_client.token_limit` | 用户配置 | 128000 | 模型上下文窗口限制 |
| `gpt_client.token_threshold` | 开发者配置 | 0.8 | 预算使用率上限 |
| `gpt_client.chunk_history_n` | 开发者配置 | 3 | 附带的最近块总结数量 |

---
最终的涉及分块的提示词结构参考
```
[系统指令 / 角色定义]
[全局信息：标题、标签、风格、格式要求]
[历史上下文区]
  <之前块的总结笔记>
  【提示词】以上是以往片段的笔记概要，供你把握整体脉络，不要在输出中重复。
[当前块指令]
  【提示词】请仅基于以下转录内容生成本段的结构化笔记，仅返回本段的总结。
[当前块转录文本 + 图片]
```
