
# BiliNote CLI 指令和参数文档

## 概述

BiliNote CLI 是一个命令行视频笔记生成工具，支持从多个视频平台（Bilibili、YouTube、抖音、快手等）自动生成 AI 笔记。

## 命令总览

| 命令 | 功能描述 |
|------|----------|
| `process` | 处理视频生成笔记（支持批量 URL 或 JSON） |
| `search` | 搜索视频并保存结果为 JSON |
| `status` | 查询任务状态 |
| `model-list` | 列出所有已配置的模型 |
| `model-set-default` | 设置默认模型 |
| `model-remove` | 删除自定义模型 |
| `config` | 管理密钥和配置 |
| `check` | 环境检查（ffmpeg、API Key、Cookie、LLM 连通性） |
| `install-shortcut` | 安装 macOS 快捷指令（仅 macOS） |
| `shortcut-prompt-off` | 关闭快捷指令安装提示 |

---

## 1. process - 处理视频生成笔记

### 功能
处理视频并生成 AI 笔记，支持单个视频、批量视频或从搜索结果 JSON 文件读取。

### 语法
```bash
bilinote process [视频URL...] [选项]
bilinote process --json <JSON文件路径> [选项]
```

### 位置参数

| 参数 | 必需 | 说明 |
|------|------|------|
| `video_urls` | 是* | 视频链接或本地文件路径（可多个，与 `--json` 二选一） |

### 选项参数

| 参数 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| `--json` | 字符串 | 无 | - | 从 search 结果 JSON 文件读取链接 |
| `--index` | 整数列表 | 无 | - | 指定 JSON 中要处理的视频序号（如 `--index 1 2 3`） |
| `--platform` | 字符串 | 自动识别 | `bilibili`, `youtube`, `douyin`, `kuaishou`, `local` | 视频平台（可选，默认自动识别） |
| `--model` | 字符串 | 配置的默认模型 | - | 模型名称（可选，默认使用配置的默认模型） |
| `--output-dir` | 字符串 | 无 | - | 批量输出目录（多任务时自动创建批次目录） |
| `--quality` | 字符串 | `medium` | `fast`, `medium`, `slow` | 音频下载质量 |
| `--screenshot` | 标志 | 关闭 | - | 在笔记中插入截图 |
| `--link` | 标志 | 关闭 | - | 在笔记中插入视频跳转链接 |
| `--style` | 字符串 | 无 | `minimal`（精简）, `detailed`（详细）, `academic`（学术）, `tutorial`（教程）, `xiaohongshu`（小红书）, `life_journal`（生活向）, `task_oriented`（任务导向）, `business`（商业风格）, `meeting_minutes`（会议纪要） | 笔记风格 |
| `--format` | 字符串列表 | `[]` | `toc`（目录）, `link`（原片跳转）, `screenshot`（原片截图）, `summary`（AI总结） | 笔记格式选项 |
| `--video-understanding` | 标志 | 关闭 | - | 启用视频多模态理解 |
| `--video-interval` | 整数 | `0` | - | 视频帧截取间隔（秒） |
| `--grid-size` | 整数列表 | 无 | - | 缩略图网格大小，如 `3 3` |
| `--no-subtitle` | 标志 | 关闭 | - | 禁用平台字幕，强制下载音频并转写 |
| `--extras` | 字符串 | 无 | - | 额外参数 |

### 示例

```bash
# 生成 B站视频笔记（自动识别平台，使用默认模型）
bilinote process https://www.bilibili.com/video/BV1xx

# 批量生成笔记
bilinote process https://www.bilibili.com/video/BV1xx https://www.bilibili.com/video/BV1xx2

# 指定模型生成笔记
bilinote process https://www.bilibili.com/video/BV1xx --model gpt-4o

# 生成 YouTube 视频笔记并插入截图
bilinote process https://youtube.com/watch?v=xxx --screenshot

# 处理本地视频
bilinote process ./video.mp4

# 从搜索结果处理全部视频
bilinote process --json ~/.bilinote/output/search_result/xxx.json

# 从搜索结果选择指定序号处理
bilinote process --json ~/.bilinote/output/search_result/xxx.json --index 1 2 3
```

### 输出
- **单任务**：同步串行执行，打印笔记预览（前500字符），保存完整笔记到文件
- **多任务**：异步并行执行，主线程串行准备 + 线程池并行 AI 处理

## 2. search - 搜索视频

### 功能
搜索视频并保存结果为 JSON 文件，可用于后续 `process` 命令批量处理。

### 语法
```bash
bilinote search <关键词> [选项]
```

### 位置参数

| 参数 | 必需 | 说明 |
|------|------|------|
| `keyword` | 是 | 搜索关键词 |

### 选项参数

| 参数 | 类型 | 默认值 | 可选值 | 说明 |
|------|------|--------|--------|------|
| `--platform` | 字符串 | `bilibili` | `bilibili`, `youtube` | 搜索平台 |
| `--output-dir` | 字符串 | `~/.bilinote/output/search_result` | - | 搜索结果保存目录 |

### 示例

```bash
# 搜索 B站视频
bilinote search "Python教程" --platform bilibili

# 搜索 YouTube 视频并指定输出目录
bilinote search "Machine Learning" --platform youtube --output-dir ./my_results
```

### 输出
- 终端显示搜索结果列表（包含标题、播放量、点赞量、收藏量、时长）
- JSON 文件保存至指定目录，文件名格式：`{时间戳}_{关键词}.json`
- JSON 结构包含 `meta`（搜索元信息）和 `results`（结果列表，带序号）

## 7. config - 管理密钥和配置

### 功能
管理 API 密钥和配置，敏感信息存入系统 keyring。

### 子命令

| 子命令 | 功能 | 参数 |
|--------|------|------|
| `set` | 设置密钥（存入系统 keyring） | `key`, `value` |
| `get` | 查看密钥（脱敏显示） | `key` |
| `delete` | 删除密钥 | `key` |
| `list` | 列出所有已知密钥及配置状态 | 无 |

### 7.1 config set

#### 语法
```bash
bilinote config set <key> <value>
```

#### 参数

| 参数 | 必需 | 说明 |
|------|------|------|
| `key` | 是 | 密钥名称（如 `DEEPSEEK_API_KEY`、`BILIBILI_COOKIE`） |
| `value` | 是 | 密钥值 |

#### 示例
```bash
bilinote config set DEEPSEEK_API_KEY sk-xxx
bilinote config set BILIBILI_COOKIE "SESSDATA=xxx; bili_jct=xxx;"
```

### 7.2 config get

#### 语法
```bash
bilinote config get <key>
```

#### 参数

| 参数 | 必需 | 说明 |
|------|------|------|
| `key` | 是 | 密钥名称 |

#### 示例
```bash
bilinote config get DEEPSEEK_API_KEY
```

### 7.3 config delete

#### 语法
```bash
bilinote config delete <key>
```

#### 参数

| 参数 | 必需 | 说明 |
|------|------|------|
| `key` | 是 | 密钥名称 |

#### 示例
```bash
bilinote config delete DEEPSEEK_API_KEY
```

### 7.4 config list

#### 语法
```bash
bilinote config list
```

#### 示例
```bash
bilinote config list
```

#### 输出
显示所有已知密钥的配置状态（脱敏显示）。

## 9. install-shortcut - 安装快捷指令（仅 macOS）

### 功能

## 配置文件说明

### 模型配置
- **敏感配置**（API Key 等）：存储在系统 keyring
- **非敏感配置**：编辑 `~/.bilinote/config.yaml`
- **自定义模型**：手动编辑 `~/.bilinote/config/models.json`

### 默认输出路径
- **笔记输出**：`~/.bilinote/output/`
- **搜索结果**：`~/.bilinote/output/search_result/`

## 完整使用流程示例

### 场景 1：单个视频笔记生成
```bash
# 1. 直接处理视频
bilinote process https://www.bilibili.com/video/BV1xx --model gpt-4o --screenshot
```

### 场景 2：批量处理搜索结果
```bash
# 1. 搜索视频
bilinote search "Python入门教程" --platform bilibili

# 2. 处理搜索结果中的全部视频
bilinote process --json ~/.bilinote/output/search_result/2026-05-19T10-30-00_Python入门教程.json

# 3. 或只处理前 3 个结果
bilinote process --json ~/.bilinote/output/search_result/2026-05-19T10-30-00_Python入门教程.json --index 1 2 3
```

### 场景 3：配置模型和密钥
```bash
# 1. 配置 API Key
bilinote config set DEEPSEEK_API_KEY sk-xxx

# 2. 查看可用模型
bilinote model-list

# 3. 设置默认模型
bilinote model-set-default gpt-4o

# 4. 使用默认模型处理视频（无需指定 --model）
bilinote process https://www.bilibili.com/video/BV1xx
```

### 场景 4：环境诊断
```bash
# 检查环境配置
bilinote check
```

## 注意事项

1. **视频链接和 --json 互斥**：不能同时使用位置参数 `video_urls` 和 `--json` 选项
2. **平台自动识别**：未指定 `--platform` 时，系统会根据 URL 自动识别平台
3. **默认模型**：未指定 `--model` 时，使用 `model-set-default` 设置的默认模型；若未设置则报错
4. **单任务 vs 多任务**：
   - 单任务：同步串行执行，保留笔记预览打印
   - 多任务：异步并行执行，主线程串行准备 + 线程池并行 AI 处理
5. **macOS 快捷指令**：仅 macOS 系统可用，安装后可从浏览器一键发送视频
6. **数量格式化**：搜索结果中使用中文单位（k、w）格式化播放量等数据
7. **时长格式化**：显示为 `1h30min`、`7min`、`45s` 等格式
