# 精简输出方案（最终版）

## 触发方式

`process` 子命令新增 `--quiet` / `-q` 参数。

**精简模式**（~4 行）：

```
✓ 已保存: /Users/xxx/BV1ZiGC6DECH.md
```

## 各项精简规则

| 输出项 | 默认模式 | 精简模式 |
|--------|---------|---------|
| 头信息 | 4 行（平台/模型/视频/分隔线） | 1 行：`📝 bilibili \| deepseek-chat \| BV1xxx` |
| yt-dlp 日志+进度条 | 正常输出 | **通过 `logger` 参数静默**，日志写入 app.log |
| 阶段提示 | 无 | 1 行：`⬇ 下载中... 🎙 转写中... 🤖 AI 生成中...` |
| 成功提示 | 3 行（分隔线+成功+路径） | 1 行：`✓ 已保存: /path/to/file.md` |
| 笔记预览 | 前 500 字符 | 不输出 |
| 错误信息 | 完整输出 | **完整输出**（不静默） |

## 核心实现：yt-dlp logger 方案

**不用 `quiet=True`**（会丢失日志），改用 `logger` 参数将 yt-dlp 输出导入项目 Python logger：

```python
import logging
logger = logging.getLogger('yt_dlp')

ydl_opts = {
    'logger': logger,  # 替代 quiet=True
    ...
}
```

yt-dlp 只调用 `debug`/`warning`/`error` 三个方法，与 Python 标准 Logger 完全兼容。项目 logger 配置 console=WARNING + file=INFO，效果：
- `debug()` → 写入 app.log，终端不显示
- `warning()` → 写入 app.log + 终端显示
- `error()` → 写入 app.log + 终端显示

## 需改动的文件

| 文件 | 改动 |
|------|------|
| `src/cli.py` | 添加 `--quiet` 参数；精简头信息/成功提示/隐藏预览 |
| `src/app/models/process_config.py` | 新增 `quiet: bool = False` |
| `src/app/downloaders/bilibili_downloader.py` | quiet 时传入 `logger` 替代默认输出 |
| `src/app/downloaders/youtube_downloader.py` | 同上 |
| `src/app/downloaders/douyin_downloader.py` | 同上 |
| `src/app/downloaders/kuaishou_downloader.py` | 同上 |
| `src/app/downloaders/xiaoyuzhoufm_download.py` | 同上 |
| `src/app/services/pipeline/preparer.py` | quiet 时打印阶段提示 |
| `src/app/services/pipeline/ai_processor.py` | quiet 时打印 AI 阶段提示（需从 PreparedTask 传 quiet） |

## 参数传递路径

```
cli.py (args.quiet)
  → ProcessConfig(quiet=True)
    → NoteGenerator.generate(cfg=cfg)
      → TaskPreparer.prepare(cfg=cfg)          # 打印阶段提示 + 传给 downloader
        → downloader.download(..., quiet=cfg.quiet)  # 传入 logger
      → AIProcessor.process(prepared)          # prepared 携带 quiet 标志
```
