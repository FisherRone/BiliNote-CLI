# 精简输出方案

## 一、当前输出分析

一次 `bilinote process` 的完整输出包含以下几类信息：

| # | 输出内容 | 来源 | 示例 | 冗余程度 |
|---|---------|------|------|---------|
| 1 | 任务头信息 | `cli.py` print | `平台: bilibili` / `模型: deepseek-chat` / `视频: URL` | 低 |
| 2 | yt-dlp 下载日志 | yt-dlp stdout | `[BiliBili] Extracting URL` / `[download] Destination` | **高** |
| 3 | yt-dlp 进度条 | yt-dlp stderr | `[download] 100% of 2.54MiB` (可能刷新多行) | **高** |
| 4 | yt-dlp 后处理日志 | yt-dlp stdout | `[ExtractAudio] Destination` / `Deleting original file` | 中 |
| 5 | 成功提示 + 文件路径 | `cli.py` print | `✓ 笔记生成成功！` / `保存到: ...` | 低 |
| 6 | 笔记预览 | `cli.py` print | markdown 前 500 字符 | 中 |
| 7 | Python logger 输出 | logging | 当前 console 级别为 WARNING，基本不输出 | 低 |

**核心问题**：yt-dlp 的输出（#2 #3 #4）占据了大部分行数，且对普通用户价值很低。

---

## 二、可精简的输出项及方案

### 2.1 yt-dlp 下载日志（优先级：高）

**现状**：yt-dlp 默认 `quiet=False`，会输出多行日志：
```
[BiliBili] Extracting URL: https://...
[BiliBili] 1ZiGC6DECH: Downloading webpage
[BiliBili] BV1ZiGC6DECH: Extracting videos in anthology
[BiliBili] 116644299021258: Extracting chapters
[info] BV1ZiGC6DECH: Downloading 1 format(s): 30280
[download] Destination: /Users/xxx/BV1ZiGC6DECH.m4a
[download] 100% of    2.54MiB in 00:00:00 at 5.75MiB/s
[ExtractAudio] Destination: /Users/xxx/BV1ZiGC6DECH.mp3
Deleting original file /Users/xxx/BV1ZiGC6DECH.m4a (pass -k to keep)
```

**精简方案**：

| 模式 | 方案 | 效果 |
|------|------|------|
| 精简模式 | 设置 `ydl_opts['quiet'] = True` | 完全静默 yt-dlp 输出 |
| 默认模式 | 保持 `quiet=False` | 保持现状 |

**实现位置**：各 downloader 文件中的 `ydl_opts` 字典，当前 `bilibili_downloader.py` 已有 `'quiet': False`，改为根据模式动态设置。

**补充**：精简模式下，用一行自定义输出替代 yt-dlp 的多行日志：
```
⬇ 下载音频中... (BV1ZiGC6DECH)
```
可在 `download()` 方法前后打印，或由 `preparer.py` 在调用前后打印。

---

### 2.2 yt-dlp 进度条（优先级：高）

**现状**：yt-dlp 的进度条在终端中会原地刷新，但如果终端不支持回车控制，或者输出被重定向，会产生大量重复行。

**精简方案**：

| 模式 | 方案 | 效果 |
|------|------|------|
| 精简模式 | `quiet=True` 直接关闭进度条 | 无进度输出 |
| 默认模式 | 保持现状 | 正常进度条 |

> 注：`quiet=True` 会同时关闭进度条和日志，无需单独处理。

---

### 2.3 任务头信息（优先级：低）

**现状**：
```
开始生成笔记...
平台: bilibili
模型: deepseek-chat
视频: https://www.bilibili.com/video/BV1ZiGC6DECH
------------------------------------------------------------
```

**精简方案**：

| 模式 | 方案 | 效果 |
|------|------|------|
| 精简模式 | 压缩为一行 | `📝 bilibili | deepseek-chat | BV1ZiGC6DECH` |
| 默认模式 | 保持现状 | 多行展示 |

---

### 2.4 笔记预览（优先级：中）

**现状**：输出 markdown 前 500 字符 + 截断提示，可能很长。

**精简方案**：

| 模式 | 方案 | 效果 |
|------|------|------|
| 精简模式 | 不输出笔记预览，只输出文件路径 | 只看到保存路径 |
| 默认模式 | 保持现状 | 输出前 500 字符预览 |

---

### 2.5 成功提示（优先级：低）

**现状**：
```
============================================================
✓ 笔记生成成功！
保存到: /Users/xxx/BV1ZiGC6DECH.md
============================================================
```

**精简方案**：

| 模式 | 方案 | 效果 |
|------|------|------|
| 精简模式 | 压缩为一行 | `✓ 已保存: /Users/xxx/BV1ZiGC6DECH.md` |
| 默认模式 | 保持现状 | 带分隔线的多行展示 |

---

### 2.6 Python logger 输出（优先级：低）

**现状**：console handler 级别为 WARNING，基本不输出到终端。无需调整。

## 四、实现方案

### 4.1 触发方式

在 `process` 子命令中新增 `--quiet` / `-q` 参数：

```python
process_parser.add_argument('--quiet', '-q', action='store_true',
                            help='精简输出模式，仅显示关键进度和结果')
```

该参数存入 `ProcessConfig`，传递到各层。

### 4.2 需要改动的文件

| 文件 | 改动 |
|------|------|
| `src/cli.py` | 添加 `--quiet` 参数；精简模式下压缩头信息和成功提示；不输出笔记预览 |
| `src/app/models/process_config.py` | 新增 `quiet: bool = False` 字段 |
| `src/app/downloaders/bilibili_downloader.py` | 根据 `quiet` 设置 `ydl_opts['quiet']` |
| `src/app/downloaders/youtube_downloader.py` | 同上 |
| `src/app/downloaders/douyin_downloader.py` | 同上（如有 yt-dlp） |
| `src/app/downloaders/kuaishou_downloader.py` | 同上（如有 yt-dlp） |
| `src/app/downloaders/xiaoyuzhoufm_download.py` | 同上（如有 yt-dlp） |
| `src/app/services/pipeline/preparer.py` | 精简模式下打印阶段提示（下载/转写/AI） |

### 4.3 quiet 参数传递路径

```
cli.py (args.quiet)
  → ProcessConfig(quiet=True)
    → NoteGenerator.generate(cfg=cfg)
      → TaskPreparer.prepare(cfg=cfg)       # 打印阶段提示 + 传给 downloader
        → downloader.download(quiet=cfg.quiet)  # 设置 yt-dlp quiet
      → AIProcessor.process()               # 无需改动
```

**关键点**：当前 `downloader.download()` 的签名中没有 `quiet` 参数，需要新增。或者更简洁地：在 `ProcessConfig` 中携带 `quiet`，由 `preparer` 在调用下载前设置 yt-dlp 选项。

### 4.4 阶段提示的实现

在 `preparer.py` 中，精简模式下在各阶段入口打印一行提示：

```python
if cfg.quiet:
    print("⬇ 下载音频中...")
# ... 调用下载

if cfg.quiet:
    print("🎙 转写中...")
# ... 调用转写
```

在 `ai_processor.py` 中：

```python
if cfg.quiet:
    print("🤖 AI 生成中...")
# ... 调用 GPT
```

> 注意：`AIProcessor` 当前不持有 `cfg`，需要将 `quiet` 传入 `process()` 方法，或通过 `PreparedTask` 传递。

### 4.5 错误输出的处理

精简模式下，错误信息**不应被静默**。当发生异常时，仍应输出完整的错误信息，确保用户能定位问题。

---

## 五、总结

| 改动项 | 精简效果 | 实现难度 |
|--------|---------|---------|
| yt-dlp quiet=True | 消除 5-10 行下载日志 | 低（改一个参数） |
| 压缩头信息 | 4 行 → 1 行 | 低 |
| 压缩成功提示 | 3 行 → 1 行 | 低 |
| 隐藏笔记预览 | 消除 N 行 | 低 |
| 添加阶段提示 | 新增 3 行进度指示 | 低 |

**总体**：精简模式可将输出从 ~20+ 行压缩到 ~4 行，改动量小，风险低。
